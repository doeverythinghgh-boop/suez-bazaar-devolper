المهمة:
تنفيذ توثيق شامل ومعياري باستخدام JSDoc لجميع الدوال، الوحدات، والمتغيرات الثابتة داخل جميع ملفات JavaScript، مع التحقق من صحة التوثيق ومطابقته للكود الفعلي.

1. أهداف التوثيق

توضيح وظيفة كل جزء من الكود دون الحاجة لقراءة التفاصيل الداخلية.

تحديد المدخلات والمخرجات بدقة لمنع أخطاء وقت التشغيل وتحسين الصيانة.

تسهيل التحويل المستقبلي إلى ES Modules.

2. إرشادات التوثيق الأساسية

يجب إضافة كتلة JSDoc قبل كل:

دالة رئيسية.

متغير ثابت مهم.

وحدة أو دالة مُصدّرة أو واسعة الاستخدام.

3. معايير توثيق الدوال

يتم استخدام العناصر التالية:

@file — اسم الملف في أعلى الملف.

@description — وصف مختصر للدالة.

@function — تعريف أنها دالة.

@param {Type} name — نوع ووظيفة المدخلات.

@returns {Type} — نوع وقيمة المخرجات.

@throws {Error} — الأخطاء المحتملة.

@see — ملفات أو دوال ذات علاقة.

4. التعامل مع الكود الميت والتكرار

للدوال أو المتغيرات غير المستخدمة: استخدام @deprecated أو @ignore مع توضيح أنها مرشحة للإزالة.

عند وجود منطق مكرر: استخدام @todo للتنبيه بضرورة دمجه في دوال Helpers.

5. التحقق من صحة التوثيق

مطابقة عدد وأنواع المدخلات مع الكود الفعلي.

مطابقة الـ return مع ما تُرجعه الدالة فعليًا.

عند وجود تناقض: تعديل التوثيق ليتوافق مع الكود.

وضع @todo للدوال غير المكتملة أو غير القابلة للتوثيق.

6. توثيق الملفات والعلاقات بينها

إضافة @file أعلى كل ملف JavaScript.

إضافة وصف يوضح علاقة الملف بالملفات الأخرى.

توثيق جميع العناصر المُصدّرة بشكل منفصل.

7. الدوال غير المتزامنة (Async)

يجب توثيق:

كون الدالة async.

ما إذا كانت تُعيد Promise.

الأخطاء التي قد ترميها عبر try/catch.

محتوى الـ Promise داخل @returns.

ملاحظة ختامية لـ Code Agent

يجب تحقيق توثيق يغطي 100% من الدوال والواجهات العامة، مع إمكانية استخراج مخطط تبعية واضح وتحسين إدارة الكود بكفاءة وأمان.
